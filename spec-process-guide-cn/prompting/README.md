## 关键原则

有效的规范开发提示遵循以下原则：

1. **具体** - 提供清晰的背景和具体示例
2. **结构化请求** - 将复杂的请求分解为可管理的部分
3. **深思熟虑地迭代** - 在先前响应的基础上构建，而不是重新开始
4. **验证理解** - 在进入下一阶段之前确认一致性

## 常见模式

- **设置背景** - 建立项目背景和约束
- **阶段转换** - 在需求、设计和任务之间平稳过渡
- **反馈整合** - 有效地采纳变更和优化
- **质量验证** - 确保输出符合您的标准
